# UX/UI Inline Entity Creation Decision

## 결정

딜 빠른 등록 중 회사/거래처/제품이 없으면, 딜 등록 모달 안에서 최소 정보로 바로 생성할 수 있게 한다.

## 이유

딜 생성에는 회사/거래처/제품 정보가 필요하다.

하지만 딜 등록 중 해당 데이터가 없다는 이유로 사용자를 다른 화면으로 보내면 입력 흐름이 끊긴다.

따라서 검색 결과가 없을 때 바로 최소 생성할 수 있게 한다.

## UX 규칙

- 검색형 선택 필드에서 기존 항목을 먼저 찾는다.
- 검색 결과가 없으면 `새 회사 만들기`, `새 거래처 만들기`, `새 제품 만들기`를 보여준다.
- 새 항목은 최소 필드만 받는다.
- 생성 후 현재 딜에 자동 선택한다.
- 자세한 정보는 해당 상세 페이지에서 보강한다.

## 금지

- 자유 텍스트만으로 회사/거래처/제품을 딜에 저장하지 않는다.
- 전체 상세 등록 폼을 딜 등록 모달 안에 넣지 않는다.
