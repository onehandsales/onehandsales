# UX/UI Inline Entity Creation Decision

## 결정

딜 빠른 등록 중 회사/담당자/제품이 없으면, 딜 등록 흐름 안에서 최소 정보로 바로 생성할 수 있게 한다.

회사/담당자/제품 생성 모달에서도 연결 또는 분류 선택지가 없으면 검색 입력값을 기준으로 바로 추가하고 자동 선택할 수 있게 한다.

## 이유

딜 생성에는 회사/담당자/제품 정보가 필요하다.

회사 생성에는 분야/지역이 필요하고, 담당자 생성에는 회사/부서/직급이 필요하며, 제품 생성에는 카테고리/상태가 필요하다.

하지만 딜 등록 중 해당 데이터가 없다는 이유로 사용자를 다른 화면으로 보내면 입력 흐름이 끊긴다.

따라서 검색 결과가 없을 때 바로 최소 생성할 수 있게 한다.

## UX 규칙

- 검색형 선택 필드에서 기존 항목을 먼저 찾는다.
- 검색 결과가 없으면 `새 회사 만들기`, `새 담당자 만들기`, `새 제품 만들기`를 보여준다.
- 새 항목은 최소 필드만 받는다.
- 생성 후 현재 딜에 자동 선택한다.
- 자세한 정보는 해당 상세 페이지에서 보강한다.
- 회사 추가 모달의 분야/지역, 담당자 추가 모달의 부서/직급, 제품 추가 모달의 카테고리/상태는 검색 결과가 없을 때 입력값으로 즉시 추가한다.
- 담당자 추가 흐름에서 회사 검색 결과가 없으면 회사 생성 흐름으로 연결하고, 생성 후 담당자 회사 필드에 자동 선택한다. 회사 생성 UI는 현재 오른쪽 문서형 패널 기준을 우선한다.
- 생성 후 자동 선택은 생성 API 응답 id에 의존하지 않고, 필요한 경우 옵션 목록을 다시 조회해 같은 이름의 항목을 찾는 방식도 허용한다.

## 금지

- 자유 텍스트만으로 회사/담당자/제품을 딜에 저장하지 않는다.
- 전체 상세 등록 폼을 딜 등록 모달 안에 넣지 않는다.
- 선택지가 필요한 필드를 사용자가 입력한 텍스트만으로 저장하지 않는다. 반드시 실제 option/entity를 생성하거나 기존 항목을 선택한다.
