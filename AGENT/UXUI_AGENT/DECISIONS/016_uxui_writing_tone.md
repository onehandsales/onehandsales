# UX/UI Writing Tone Decision

## 결정

`한손에 영업 / onehand.sales`의 사용자 노출 문구는 해요체를 기본으로 한다.

Toss UX Writing Guide의 원칙을 참고해, 제품 문구는 쉽고 짧고 능동적으로 작성한다.

## 적용 원칙

- `입니다/습니다/합니다` 체를 쓰지 않는다.
- 빈 상태, 오류, 제한 안내에서는 `없습니다`, `할 수 없습니다`보다 사용자가 다음에 할 수 있는 행동을 먼저 보여준다.
- `저장되었습니다` 같은 수동형보다 `저장했어요` 같은 능동형을 쓴다.
- `~시겠어요?`, `시나요?`, `께`, `계시다`, `여쭈다`처럼 과한 경어를 피한다.
- 한 문장에는 하나의 의미만 담고, 가능한 한 한 줄로 말한다.
- 버튼은 짧은 행동형으로 쓰고, 다이얼로그 왼쪽 버튼은 `닫기`로 통일한다.
- 메뉴, 테이블 헤더, 필터명, 도메인명은 짧은 명사형을 허용한다.

## 이유

이 제품은 반복해서 쓰는 개인 영업 업무 도구다.

딱딱한 `습니다` 체나 부정형 문구가 많으면 화면이 무겁고 시스템 중심으로 느껴진다. 해요체, 능동형, 긍정형 문구를 쓰면 사용자가 상태를 더 빨리 이해하고 다음 행동으로 이동하기 쉽다.

## 적용 예

| 기존 | 변경 |
|---|---|
| 불러오는 중입니다. | 불러오고 있어요. |
| 등록된 회사가 없습니다. | 새 회사를 등록하면 여기에서 볼 수 있어요. |
| 요청을 처리할 수 없습니다. | 잠시 후 다시 시도해 주세요. |
| 저장되었습니다. | 저장했어요. |
| 회사명을 입력해주세요. | 회사명을 입력해 주세요. |
