# UX/UI Create Flow Decision

## 결정

회사/담당자/제품/딜 등록은 다음 구조로 간다.

- 빠른 등록: 모달
- 상세 등록/수정: 별도 상세 페이지

## 이유

영업자는 현장에서 정보를 빠르게 남겨야 한다.

처음부터 모든 정보를 입력하게 하면 등록을 미루거나 누락할 가능성이 높다.

반대로 딜 상세, 회사 히스토리, 제품 연결, Memo 기록 같은 복잡한 정보는 모달에서 다루기 어렵다.

따라서 최소 필드로 빠르게 생성하고, 상세 페이지에서 보강한다.

## 빠른 등록 원칙

- 현재 화면 맥락을 유지한다.
- 필수 또는 거의 필수인 필드만 받는다.
- 저장 후 상세로 이동할 수 있어야 한다.
- 복잡한 입력은 상세 페이지로 넘긴다.

## 빠른 등록 최소 필드

Company:

- 회사명
- 카테고리/업종 optional
- initial Memo optional

Contact:

- 이름
- 회사 연결
- 전화번호 optional
- 부서/직책 optional

Product:

- 제품명
- 카테고리 optional
- 단가 optional

Deal:

- 딜이름
- 회사
- 담당자
- 제품
- 금액
- 단계
- 가능성
- 다음 행동 optional
- 마감/예상 종료일 optional

## 딜 등록 중 없는 항목 생성

딜 빠른 등록 중 회사/담당자/제품이 없으면 모달 안에서 최소 정보로 바로 생성한다.

동작:

- 검색형 선택 필드에서 기존 항목을 먼저 찾는다.
- 검색 결과가 없으면 새 항목 만들기 액션을 보여준다.
- 새 항목은 최소 필드만 입력받는다.
- 생성된 항목은 현재 딜에 자동 선택된다.

금지:

- 회사/담당자/제품을 자유 텍스트로만 딜에 저장하지 않는다.
- 전체 상세 등록 폼을 딜 등록 모달 안에 넣지 않는다.
