# UX/UI Home Screen Decision

## 결정

로그인 후 첫 화면은 딜 파이프라인 중심으로 한다.

## 이유

이 앱의 핵심 가치는 개인 영업자가 현재 진행 중인 딜을 빠르게 파악하고 다음 행동을 결정하는 것이다.

따라서 홈은 일반 대시보드, 빠른 입력, 통합 검색보다 딜 파이프라인을 먼저 보여준다.

## UX 방향

홈에서 가장 먼저 보여줄 것:

- 진행 중인 딜
- 단계별 딜 상태
- 딜 금액
- 가능성 상태
- 다음 행동/follow-up
- 마감 임박 또는 지연 상태

보조 영역:

- 오늘 일정
- 최근 활동
- 최근 회의록
- 빠른 생성 버튼
- 알림

## 참고 UI 방향

참고 파일:

- `example.html`

이 파일은 구현 기준이 아니라 UX/UI 방향 참고용이다.

채택할 방향:

- 좌측 사이드바
- 상단 페이지 타이틀과 주요 액션
- 단계 탭
- 검색/필터
- 리스트/테이블 중심 딜 관리
- 선택 딜 우측 상세 패널
- 활동 로그/메모를 상세 패널에서 빠르게 확인

조정할 방향:

- 브랜드는 `SalesKit`이 아니라 `한손에 영업 / onehand.sales`를 사용한다.
- `오프더레코드` 표현은 사용하지 않고 `개인 메모`를 사용한다.
- 가능성 기본 UI는 숫자 퍼센트가 아니라 `긍정 / 중립 / 부정`이다.
- 베이지/크림 계열이 지배적인 화면은 피한다.
- 토스 UX/UI처럼 심플하고 명확하게 가되, 영업자가 딜을 비교할 수 있는 정보 밀도는 유지한다.
