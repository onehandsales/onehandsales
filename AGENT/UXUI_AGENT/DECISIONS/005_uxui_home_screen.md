# UX/UI Home Screen Decision

## 결정

`/`는 공개 진입면으로 두고, 로그인 후 첫 화면은 `/app` 홈 대시보드로 한다. 핵심 딜 작업은 `/app/deals` 딜 파이프라인에서 처리한다.

현재 구현 상태:

- 2026-07-09 기준 `/app` 홈은 Schedule/Deal/MeetingNote API 조합 대시보드다.
- 홈은 오늘 일정, 진행 딜, 마감 임박, 최근 회의록, 빠른 실행, 최근 활동을 표시한다.
- 딜 파이프라인 경험은 `/app/deals`에서 운영한다.
- 딜 목록/상세/생성의 고밀도 작업은 `/app/deals`를 기준으로 유지한다.
- 상단 통합검색은 Backend `GET /api/search`와 User Web GlobalSearch 연결 기준으로 구현 완료 상태다.

## 이유

이 앱의 핵심 가치는 개인 영업자가 현재 진행 중인 딜을 빠르게 파악하고 다음 행동을 결정하는 것이다.

따라서 홈은 오늘 해야 할 일과 최근 맥락을 빠르게 보여주고, 딜 비교/수정/생성은 `/app/deals`에서 더 높은 밀도로 제공한다.

## UX 방향

홈에서 가장 먼저 보여줄 것:

- 오늘 일정
- 진행 중인 딜
- 단계별 딜 상태
- 딜 금액
- 다음 행동/follow-up
- 마감 임박 또는 지연 상태
- 최근 회의록

보조 영역:

- 최근 활동
- 빠른 생성 버튼
- 알림

## 참고 UI 방향

참고 파일:

- `example.html`

이 파일은 구현 기준이 아니라 UX/UI 방향 참고용이다.

채택할 방향:

- 좌측 사이드바
- 상단 페이지 타이틀과 주요 액션
- 단계 탭
- 검색/필터
- 리스트/테이블 중심 딜 관리
- 선택 딜 우측 상세 패널
- 활동 로그/메모를 상세 패널에서 빠르게 확인

조정할 방향:

- 브랜드는 `SalesKit`이 아니라 `한손에 영업 / onehand.sales`를 사용한다.
- `오프더레코드` 표현은 사용하지 않고 `Memo 기록`을 사용한다.
- 가능성 기본 UI는 숫자 퍼센트가 아니라 `긍정 / 중립 / 부정`이다.
- 베이지/크림 계열이 지배적인 화면은 피한다.
- 토스 UX/UI처럼 심플하고 명확하게 가되, 영업자가 딜을 비교할 수 있는 정보 밀도는 유지한다.
