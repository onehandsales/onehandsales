# UX/UI Admin Tone Decision

## 결정

Admin Web은 같은 블루/그레이 신뢰 톤을 쓰되 User Web보다 정보 밀도를 높인다.

## 원칙

- 데스크톱 전용
- 테이블 중심
- 필터 중심
- 운영 콘솔 느낌
- 민감정보 기본 마스킹
- 위험 액션은 사유 입력과 감사 로그 필수

## 이유

Admin은 사용자 앱처럼 빠르고 부드러운 입력 경험보다 정확성, 추적성, 안전성이 더 중요하다.

따라서 User Web보다 업무툴에 가까운 밀도와 명확한 테이블/필터 구조를 사용한다.
