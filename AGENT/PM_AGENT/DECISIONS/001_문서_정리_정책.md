# 문서 정리 정책

## 결정

기존 문서는 필요한 내용만 `AGENT` 정본 문서로 흡수하고, 나머지는 삭제하지 않고 archive로 이동한다.

## 정책

- `AGENT` 아래 문서를 앞으로의 정본으로 사용한다.
- 기존 루트 문서, `need_study`, `notion_log`는 정본이 아니다.
- 기존 문서에서 필요한 내용은 다음 폴더로 재작성/흡수한다.
  - 제품 기획: `AGENT/PM_AGENT/PLANNING`
  - UX/UI 기획과 결정: `AGENT/UXUI_AGENT`
  - Backend 기술/아키텍처/코드/주석/로깅 규칙: `AGENT/SOFTWARE_AGENT/BACKEND_AGENT`
  - Frontend 기술/아키텍처/코드/주석/로깅 규칙: `AGENT/SOFTWARE_AGENT/FRONT_AGENT`
  - DB schema: `AGENT/SOFTWARE_AGENT/DB_SCHEMA`
  - 제품/운영 결정: `AGENT/PM_AGENT/DECISIONS`
  - Backend 기술 결정: `AGENT/SOFTWARE_AGENT/BACKEND_AGENT/DECISIONS`
  - Frontend 기술 결정: `AGENT/SOFTWARE_AGENT/FRONT_AGENT/DECISIONS`
  - UX/UI 결정: `AGENT/UXUI_AGENT/DECISIONS`
- 흡수 완료된 기존 문서는 삭제하지 않고 archive로 이동한다.

## Archive 위치

- archive 위치는 `archive`로 한다.
- 기존 루트 문서, `need_study`, `notion_log` 등은 흡수 후 `archive` 아래로 이동한다.

## Archive 원칙

- archive는 참고용이며 새 작업의 기준으로 삼지 않는다.
- archive 문서가 정본과 충돌하면 `AGENT` 문서를 우선한다.
- archive 이동 전에는 어떤 내용이 어디로 흡수됐는지 기록한다.
