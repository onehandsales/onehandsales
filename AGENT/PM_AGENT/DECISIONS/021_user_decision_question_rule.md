# 사용자 결정 질문 진행 규칙

## 1. 결정

사용자의 판단이 필요한 미결정 항목이 여러 개 있을 때는 한 번에 하나씩 질문한다.

각 질문은 단순히 "어떻게 할까요?"라고 묻지 않고, 사용자가 바로 의사결정할 수 있도록 다음 정보를 함께 제시한다.

- 결정해야 하는 항목
- 질문의 의미
- 왜 지금 결정해야 하는지
- 선택지
- 선택지별 영향
- 추천안
- 추천 이유
- 사용자가 답한 뒤 반영할 문서 위치

사용자는 `A`, `B`, `C`처럼 짧게 답할 수 있어야 하며, 직접 작성한 다른 답변도 허용한다.

## 2. 이유

미결정 항목을 한 번에 많이 제시하면 사용자가 각 항목의 영향 범위를 다시 해석해야 한다.

특히 인증, 보안, 운영, 배포, 외부 Provider, 민감정보처럼 제품과 구현에 모두 영향을 주는 결정은 질문 자체에 판단 근거가 포함되어야 한다. 그래야 사용자는 빠르게 결정하고, AI는 사용자의 의도를 임의로 확정하지 않는다.

## 3. 적용 범위

이 규칙은 다음 상황에 적용한다.

- `TODO/{PLAN_NAME}/COMMON/G00-PENDING-QUESTIONS.md` 같은 선행 결정 질문을 처리할 때
- 기획 검토 결과에서 `Question` 또는 `보류` 항목이 남았을 때
- 보안, 권한, 결제, 운영, 배포, 외부 Provider, 비용, 민감정보 범위처럼 사용자의 판단이 필요한 항목을 발견했을 때
- AGENT 또는 TODO 문서 간 충돌을 사용자의 결정으로 해소해야 할 때

## 4. 진행 방식

질문은 다음 순서를 따른다.

1. 가장 먼저 결정해야 하는 항목 하나를 고른다.
2. 질문의 의미와 지금 결정해야 하는 이유를 설명한다.
3. 선택지를 2개 이상 제시한다.
4. 추천안을 하나 제시하고 이유를 설명한다.
5. 사용자가 답하면 관련 AGENT 또는 TODO 문서에 즉시 반영한다.
6. 반영 후 다음 미결정 항목으로 넘어간다.

사용자가 한 번에 여러 항목을 처리하라고 명시하지 않는 한, 독립적인 질문 여러 개를 한 메시지에 묶지 않는다.

## 5. 금지

- 질문만 던지고 판단 기준을 생략하지 않는다.
- 여러 미결정 항목을 한꺼번에 던져 사용자가 다시 정리하게 만들지 않는다.
- 사용자의 결정이 필요한 항목을 추천안만으로 임의 확정하지 않는다.
- 사용자가 답한 결정을 문서에 반영하지 않은 채 다음 질문으로 넘어가지 않는다.
