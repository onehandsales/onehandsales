# AGENT

이 폴더는 프로젝트의 기획, 아키텍처, 컨벤션, 결정 기록을 담는 정본 문서 공간이다.

## 폴더

- `PLANNING`: 제품 기획, PRD, MVP 범위, 사용자 흐름
- `CONVENTION`: 코드, 네이밍, 주석, 로깅, 문서 작성 규칙
- `ARCHITECTURE`: 프론트엔드, 백엔드, Admin, 테스트, 이후 모바일 아키텍처 규칙
- `DECISIONS`: 확정 결정과 미정 결정 기록

## 정본 문서

새 작업을 시작할 때는 아래 문서를 먼저 확인한다.

- `DECISIONS/000_확정_결정.md`
- `PLANNING/PRD.md`
- `PLANNING/MVP_SCOPE.md`
- `PLANNING/DATA_MODEL.md`
- `PLANNING/USER_FLOW_AND_SCREENS.md`
- `ARCHITECTURE/OVERVIEW.md`
- `ARCHITECTURE/BACKEND.md`
- `ARCHITECTURE/FRONTEND_USER_WEB.md`
- `ARCHITECTURE/ADMIN_WEB.md`
- `ARCHITECTURE/TESTING.md`
- `CONVENTION/BACKEND.md`
- `CONVENTION/FRONTEND_USER_WEB.md`
- `CONVENTION/ADMIN_WEB.md`
- `CONVENTION/COMMENT_AND_LOGGING.md`
- `CONVENTION/DOCUMENTATION.md`

archive 또는 legacy 문서가 이 폴더의 문서와 충돌하면 `AGENT` 문서를 우선한다.

모바일 아키텍처는 아직 정본이 아니다. 모바일 개발을 시작할 때 별도 문서로 추가한다.
