# PMF 검증 (Product-Market Fit Validation)

> 이 문서는 **"무엇을 만들지"가 아니라 "만들기 전에/만들면서 무엇을 검증할지"**에 대한 것.
>
> [06번 추가_권장_작업](06_추가_권장_작업.md)이 **What** (기능 후보)이라면,
> 이 문서(09)는 **Whether/Who/Why** (시장이 있는가, 누구를 위한 건가, 왜 돈을 낼 건가).
>
> **솔직한 출발점**:
> 현 기획은 **반도체 영업 경험자 1명 + 백엔드 개발자 1명**의 대화에서 출발했다.
> n=1 페인포인트로 만든 제품은 n=1 시장에서만 PMF가 난다.
> → 출시 전/베타 단계에서 **다른 산업 영업사원과 대조**해야 한다.
>
> 작성일: 2026-05-21

---

## 0. 이 문서의 사용법

1. **출시 전 (Week 1~2)**: 5장 인터뷰 가이드로 5명 인터뷰. 결과를 가설(2장)과 대조.
2. **베타 진입 시 (Week 4)**: 6장 측정 지표를 모니터링 셋업에 포함.
3. **베타 4주 후**: 7장 Kill/Pivot/Persevere 기준으로 판단.

각 가설/검증 활동은 ✅ 검증됨 / ❌ 반증됨 / ⏸ 미검증 으로 추적.

## 1. 현 기획의 PMF 가설 명시화

> 현 기획은 아래 가설들이 **모두 참**이라는 전제 위에 서 있다. 하나라도 거짓이면 PMF는 어렵다.

### H-01. 영업사원 개인은 회사 CRM과 별개로 본인용 도구를 원한다
- **근거**: 통화 원본 — "회사 양식과 무관한 본인 정리"
- **검증 필요**: 다른 산업도 같은가? 보험 설계사는 회사 CRM 안에서 끝낼 수 있는가?
- **반증 신호**: 다른 산업 영업사원이 "회사 CRM이면 충분"이라고 답하면 시장 좁아짐

### H-02. 노트북↔모바일 동기화는 결정적 페인포인트다
- **근거**: 통화 원본 — "운전 중 핸드폰에서 못 봄"
- **검증 필요**: 노션/에버노트/구글킵으로 이미 해결되지 않았는가? 영업사원이 그걸 안 쓰는 이유는?
- **반증 신호**: "노션 쓰는데요" 답이 5명 중 3명+

### H-03. off-the-record 메모는 차별화 가치다
- **근거**: 통화 원본 — "OO 부서에 OO 있음 기억하는 게 중요"
- **검증 필요**: 다른 영업사원도 이런 메모를 하는가? 어디에 적는가?
- **반증 신호**: "그냥 카톡 나에게 보내요" 또는 "메모장 앱이면 충분"

### H-04. 명함 OCR이 핵심 기능이다
- **근거**: 통화 원본 — "사진 찍으면 자동 저장"
- **검증 필요**: 리멤버를 이미 쓰는 사람이 얼마나 되는가? 안 쓰는 이유는?
- **반증 신호**: "리멤버 쓰는데 만족함" 답이 다수

### H-05. 영업사원은 새 도구 학습에 시간을 쓸 의향이 있다
- **검증 필요**: 일평균 15분+ 새 도구 학습할 동기가 있는가?
- **반증 신호**: "지금도 시간 없는데 새 앱 또 익히기 싫어요"

### H-06. 무료로 가치를 느낀 사용자는 유료로 전환할 의향이 있다
- **현재 기획**: 수익화 미정 ([00_INDEX.md](00_INDEX.md))
- **검증 필요**: 어떤 기능에 돈 낼 의향이 있는가?
- **반증 신호**: 베타 사용자에게 "월 5천원 낼 의향?" 물었을 때 0명

### H-07. 첫 100명 사용자를 어디서 데려올지 채널이 있다
- **현재 기획**: GTM 채널 미정
- **검증 필요**: 출시일에 베타 사용자 5~10명을 어디서 모을 것인가?
- **반증 신호**: 베타 사용자 모집에 2주 이상 걸림

## 3. 경쟁 분석

### 3.1 직접 경쟁

| 제품 | 강점 | 약점 | 우리와의 격차 |
|------|------|------|-------------|
| **리멤버** | 명함 OCR 한국 1위, 700만+ 사용자, 무료, 비즈니스 네트워킹 | 영업 파이프라인 약함, off-the-record 메모 없음 | **명함은 우리가 못 이김**. 차별점은 영업건+메모+거래처 통합 |
| **세일즈포스 모바일** | 글로벌 표준 CRM | 무겁고 비쌈, 회사용, 한국화 부족 | 정면 경쟁 아님 (B2B vs B2C) |
| **구글 연락처 + 노션** | 무료, 검증됨 | 영업 워크플로 X, 명함 OCR X | 우리가 영업 특화로 이김 |
| **네이버 웍스 등 협업툴** | 회사 자체 솔루션 | 개인용 X | 정면 경쟁 아님 |

### 3.2 간접 경쟁

| | 무엇 | 영업사원이 실제 쓰는 이유 |
|---|------|---------------------|
| **엑셀** | 단순/유연 | "익숙해서". 이게 가장 큰 적 |
| **카톡 나에게 보내기** | 빠른 메모 | 0초에 메모 가능 |
| **종이 수첩** | 의외로 많음 | 운전 중에도 OK |
| **회사 CRM (있는 경우)** | 강제 사용 | 회사 정책 |

### 3.3 차별화 가능 영역 (현재 기획 기준)

| 영역 | 차별점 | 강도 |
|------|--------|------|
| 명함 OCR | 리멤버보다 좋게? | 🔴 약함 (이길 확률 낮음) |
| 영업건(Deal) 파이프라인 | 리멤버에 없음 | 🟢 강함 |
| **off-the-record 메모** | 통화에서 핵심 — 리멤버/엑셀에 없는 영역 | 🟢 강함 |
| 노트북-모바일 동기화 | 노션 대비? | 🟡 모호 |
| 엑셀 Import/Export | 엑셀 사용자 흡수에 결정적 | 🟢 강함 (전환 비용 0) |

**전략적 결론**: "명함앱"으로 포지셔닝하면 진다. **"영업 인사이트 노트"**로 포지셔닝하면 빈 영역.

## 4. 타겟 좁히기 (Niche Down)

### 4.1 현재 "영업사원"은 너무 넓다

| 산업 | 워크플로 특성 | 우리 제품 적합도 |
|------|-------------|--------------|
| **반도체 부품 B2B** (기획 원천) | 거래처 ~150, 영업건 길고 복잡, off-the-record 인맥 중요 | 🟢 매우 적합 |
| 제약 MR | 회사 CRM 강제, 의사 방문 패턴 정형화 | 🟡 모호 |
| 보험 설계사 | 회사 시스템 안에서 일함, 개인 도구 여지 적음 | 🔴 부적합 |
| 부동산 중개 | 매물 중심, 거래처 관리는 부차적 | 🟡 모호 |
| 식자재/주류 영업 | 거래처 ~50, 반복 주문, 단가 정보 중요 | 🟢 적합 |
| 산업재 영업 (기계/소재/화학) | 반도체와 유사 패턴 | 🟢 적합 |
| 광고/매체 영업 | 거래처 적고 미팅 중심 | 🟡 모호 |
| 자동차 영업 | 회사 시스템 강제 | 🔴 부적합 |

### 4.2 권고 — "B2B 산업재 영업사원" 우선 타겟

**왜**:
- 반도체 영업 = 산업재의 한 종류, 기획 원천과 일치
- 거래처 100~300명 규모 = 엑셀 한계 초과, CRM 필요
- 회사 양식보다 본인 정리가 필요한 빈도 높음
- off-the-record 인맥 정보의 가치 큼

**구체화**:
- 첫 베타: 반도체/디스플레이/화학소재/산업기계 영업사원 10명
- 작성자(반도체 영업) 본인 네트워크에서 시드
- "친구의 친구"까지 2단계 확장

**거절할 타겟** (지금은):
- 보험/자동차/금융 — 회사 시스템 강제 영역
- B2C 영업 (방판/홈쇼핑) — 워크플로 다름

## 5. 출시 전 검증 (인터뷰 가이드)

### 5.1 인터뷰 대상

- **목표**: 5~7명, 다른 산업 영업사원
- **확보 방법**: 작성자 지인 네트워크, 링크드인, 산업 카페
- **시간**: 30분/명, 화상 또는 전화 OK

### 5.2 인터뷰 스크립트

> 답변자가 영업 도구 얘기를 자연스럽게 하도록. **우리 제품 설명 절대 먼저 하지 말 것.**

**① 워크플로 (10분)**
- "하루 영업 활동을 시간 순서대로 알려주세요"
- "거래처/고객 정보를 어디에 적으세요? 보여줄 수 있나요?"
- "최근 1주일 동안 영업 데이터 때문에 짜증났던 순간이 있었어요?"

**② 도구 현황 (10분)**
- "지금 쓰는 영업 도구를 다 알려주세요" (CRM/엑셀/노션/카톡/메모/리멤버 등)
- "그중에서 가장 자주 여는 건요? 왜요?"
- "회사 CRM이 있나요? 있다면 만족도는?"
- "리멤버 쓰세요? 왜 쓰세요/안 쓰세요?"

**③ 페인포인트 (5분)**
- "지금 영업 도구에서 가장 불편한 점 3가지만"
- "그 불편함을 어떻게 해결하고 계세요?"
- "마법의 도구가 생긴다면 무엇을 해줬으면?"

**④ 지불 의향 (5분 — 마지막에만)**
- "지금 쓰는 영업 도구에 한 달에 얼마 쓰고 계세요?"
- "[우리 제품 1분 설명] — 이런 게 있다면 한 달에 얼마 낼 의향?"
- "0원이라면 왜요?"

### 5.4 가설 검증 결과 표기

각 인터뷰 후 H-01 ~ H-07 가설을 다음으로 표기:
- 🟢 **지지** (응답이 가설을 강화)
- 🔴 **반증** (응답이 가설을 부정)
- ⚪ **무관**

**판단 기준**:
- 같은 가설에 🟢가 5명 중 3명+ → 가설 유효
- 🔴가 5명 중 3명+ → 가설 위험, 재검토 필수
- 혼재 → 더 인터뷰

## 6. 베타 단계 측정 지표 (출시 후 4주)

### 6.1 활성도 지표

| 지표 | 정의 | 의미 있는 임계값 |
|------|------|--------------|
| **D1 Retention** | 가입 다음 날 재방문 비율 | > 40% |
| **D7 Retention** | 가입 7일 후 재방문 비율 | > 25% |
| **W4 Retention** | 가입 4주 후 주간 사용 | > 20% |
| **DAU/MAU** | 일평균 사용자 / 월평균 | > 0.2 |

### 6.2 행동 지표 (PMF 결정적)

| 지표 | 정의 | PMF 신호 |
|------|------|----------|
| **거래처 등록 수 (사용자당 중앙값)** | 가입 후 1주일 내 | 10개+ |
| **명함 OCR 사용 횟수 (사용자당)** | 첫 주 | 3회+ |
| **메모 작성 빈도** | 사용자당 주당 | 5회+ |
| **노트북→모바일 교차 사용 비율** | 둘 다 쓰는 사용자 / 전체 | > 30% |
| **30일 후에도 주 1회+ 사용** | "정착" 지표 | > 15% |

### 6.3 자기 보고 지표 (정기 NPS/Survey)

**Sean Ellis Test** (PMF 황금 질문):
> "이 제품을 더 이상 쓸 수 없다면 어떨 거 같아요?"

선택지:
- 🔴 매우 실망스러울 것 같다
- 🟡 좀 실망스러울 것 같다
- ⚪ 실망스럽지 않을 것 같다
- ⚫ 이미 안 씀

**PMF 임계값**: 🔴 답변이 **40% 이상**.

언제 측정: 가입 후 2주, 4주, 8주.

### 6.4 측정 인프라

- 백엔드 행동 로그 ([06번 C-09](06_추가_권장_작업.md)와 연결)
- Sentry는 에러만 — 별도 분석 도구 검토 (Plausible, PostHog, Mixpanel)
- 베타 사용자에게 Survey는 인앱 ([06번 C-10](06_추가_권장_작업.md))

## 7. Kill / Pivot / Persevere 기준

> 베타 4~8주 후 다음 기준으로 결정.

### 7.1 🟢 Persevere (계속 간다)

다음 중 **3개 이상**:
- Sean Ellis Test 🔴 응답 > 40%
- W4 Retention > 25%
- 사용자 N명이 "친구에게 추천했다" 답변
- 인터뷰에서 "이거 없으면 곤란해요" 답변 다수
- 일평균 사용 시간 > 5분

→ Phase 4 진행 + 수익화 가설 실험 시작

### 7.2 🟡 Pivot (방향 전환)

다음 중 **2개 이상**:
- 활성도는 있는데(D7 > 25%) Sean Ellis 점수 낮음 → "있어도 그만 없어도 그만"
- 특정 산업/규모에서만 사용 집중 → **타겟 좁히기 피벗**
- 핵심 기능(명함 OCR)은 안 쓰고 다른 기능(메모)만 씀 → **포지셔닝 피벗**
- 사용자가 우리가 의도 안 한 방식으로 씀 → **유스케이스 피벗**

피벗 옵션 예시:
- 🔄 **타겟 좁히기**: "B2B 산업재 영업사원만" 마이크로 SaaS
- 🔄 **기능 좁히기**: 명함 OCR 빼고 "영업 인사이트 노트"로 재포지셔닝
- 🔄 **B2B로 전환**: 개인이 안 되면 회사 단위 (단, [D-001](04_의사결정_기록.md) 정면 위배)

### 7.3 🔴 Kill (중단 검토)

다음 중 **2개 이상**:
- 베타 사용자 모집 자체가 안 됨 (4주 동안 10명 미만)
- D7 Retention < 10%
- Sean Ellis Test 🔴 < 10%
- 활성 사용자 인터뷰에서 "리멤버/엑셀이면 충분" 답변이 다수
- 유료 의향 응답 0건

→ 중단 또는 다른 문제로 피벗.
→ **이때 가장 중요한 건 자존심 내려놓기**. "고깃집 김치찌개" 원칙은 자기 제품에도 적용된다.

## 8. 우리가 가진 비대칭 우위

> 같은 시장에 뛰어드는 다른 팀 대비 **우리만 가진 것**.

| 우위 | 활용법 |
|------|--------|
| **반도체 영업 실전 경험** | 도메인 디테일이 압도적. 리멤버/세일즈포스는 영업 현장 모름 |
| **2인 풀스택 + AI** | 빠른 반복 (4주 출시), 사용자 인터뷰 직접 가능 |
| **수익화 압박 없음** (단기) | 광고/UX 더럽히지 않고 가치 추구 가능 |
| **작성자의 영업 네트워크** | 베타 시드 사용자 확보 채널 — 이게 H-07 답이 될 수 있음 |

**위험**: 위 우위들은 "오래가지 않음". 6개월 후 시장이 알아채면 거대 플레이어가 동일 기능 추가 가능. 그 안에 lock-in (사용자가 못 떠나는 이유) 만들어야.

## 10. PMF 단계별 체크리스트

> Marc Andreessen의 PMF 정의: *"You can always feel when product/market fit isn't happening."*

### 10.1 PMF 못 본 상태의 신호
- 사용자가 가입은 하는데 활성화는 안 됨
- 인터뷰에서 "좋은 거 같아요" (구체적 가치 묘사 X)
- 무료여도 이탈
- 입소문 안 남

### 10.2 PMF 본 상태의 신호
- 사용자가 알아서 친구에게 추천
- "이거 없으면 일 못 해요" 같은 답변
- 사용자가 기능 요청을 적극적으로 함 (= 의존 시작)
- 가입자가 알아서 늘어남 (광고 없이도)
