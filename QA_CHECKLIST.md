# onehand.sales QA 체크리스트

작성일: 2026-07-06

이 문서는 `BE`, `FE`, `AGENT` 문서를 기준으로 만든 QA 체크리스트입니다. QA를 처음 진행하는 사람이 그대로 따라가며 확인할 수 있도록 기능 QA와 UX/UI QA를 함께 정리합니다.

현재 QA 대상 제품은 `Web + 반응형 Web + 모바일 브라우저 Web`입니다. iOS/Android 네이티브 앱은 현재 구현/QA 범위가 아니며, 모바일 QA는 모바일 브라우저에서 현재 Web 제품의 핵심 업무가 사용 가능한지 확인하는 기준으로 진행합니다.

## 1. QA 목적

QA는 제품이 의도한 대로 동작하는지, 사용자가 실제 업무를 끝까지 수행할 수 있는지, 데이터가 안전하게 저장되는지 확인하는 과정입니다.

이번 QA의 핵심 질문은 다음과 같습니다.

- 사용자가 로그인 후 핵심 영업 업무를 끝까지 처리할 수 있는가?
- 회사, 담당자, 제품, 딜, 일정, 회의록 데이터가 정확하게 생성/조회/수정/삭제/복구되는가?
- 화면이 데스크톱, 태블릿, 모바일 브라우저에서 깨지지 않고, 사용자가 다음 행동을 이해할 수 있는가?
- 잘못된 입력, 서버 오류, 권한 오류, 삭제된 데이터 접근 같은 예외 상황에서 앱이 무너지지 않는가?
- 현재 구현 범위가 아닌 기능이 사용자에게 잘못 노출되지 않는가?

## 4. QA 결과 표기법

각 항목은 아래 상태 중 하나로 기록합니다.

- `[PASS]`: 기대한 대로 동작함
- `[FAIL]`: 기대 결과와 다르게 동작함
- `[BLOCKED]`: 환경, 계정, 외부 API, 데이터 문제 때문에 확인 불가
- `[N/A]`: 현재 제품 범위가 아니거나 이번 QA 제외 범위
- `[NEEDS CHECK]`: 애매해서 기획/개발 확인 필요

버그를 발견하면 반드시 아래 4가지를 남깁니다.

- 재현 경로
- 기대 결과
- 실제 결과
- 화면 캡처 또는 콘솔/API 에러

## 5. 심각도 기준

- `S0 Blocker`: 로그인 불가, 앱 실행 불가, 핵심 데이터 전체 접근 불가
- `S1 Critical`: 데이터 손실, 다른 사용자 데이터 노출, 결제/권한/보안급 문제
- `S2 Major`: 핵심 업무 흐름 실패, 생성/수정/삭제/복구 실패
- `S3 Minor`: 일부 화면/필터/문구/정렬 문제, 우회 가능
- `S4 Polish`: 시각적 어색함, 문구 다듬기, 정렬 간격 개선
